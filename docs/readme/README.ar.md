<p align="center" dir="rtl">
  كشف LSB، التحليل الإحصائي بمربع كاي، تحليل RS، فحص DCT الجنائي، إخفاء البيانات في الصوت، ترميز النصوص بأحرف عديمة العرض، التحليل الجنائي للملفات، كشف الملفات المتعددة الصيغ، تحديد الترميز، إخفاء الفيديو والشبكة والأرشيفات، تحليل JPEG المتقدم، الطيف الموزع، BPCS، إنشاء وتضمين البيانات المخفية &mdash; كل ذلك موحّد في خادم MCP واحد.<br>

## المشكلة

الإخفاء الرقمي (Steganography) هو فن إخفاء البيانات في مرأى الجميع &mdash; داخل الصور، الملفات الصوتية، المستندات، وحتى نصوص يونيكود. يُستخدم في مسابقات CTF، التحقيقات الجنائية الرقمية، قنوات الاتصال السرية، وحمولات البرمجيات الخبيثة. يتطلب اكتشافه مزيجًا من التحليل الإحصائي، تحليل الصيغ المتخصصة، قياس الإنتروبيا، والخبرة المتخصصة.

</div>

```
سير عمل تحليل الإخفاء الرقمي التقليدي:
  كشف إخفاء الصور              ->  zsteg + stegsolve (أداتان، Ruby + Java)
  تحليل مربع كاي                ->  سكريبت Python مخصص
  تحليل RS                      ->  كود MATLAB/Python مخصص
  فحص DCT لصور JPEG              ->  stegdetect (أداة C متوقفة منذ 2004)
  استخراج بيانات LSB             ->  zsteg + steghide + openstego (3 أدوات)
  إخفاء البيانات في الصوت        ->  Audacity يدوي + سكريبتات مخصصة
  كشف النصوص عديمة العرض          ->  أدوات ويب + فحص يدوي
  فحص الملفات الجنائي / binwalk   ->  binwalk + foremost + xxd (3 أدوات)
  بيانات EXIF الوصفية            ->  exiftool (يتطلب Perl)
  كشف الترميز                    ->  واجهة CyberChef + تخمين يدوي
  ─────────────────────────────────
  المجموع: أكثر من 10 أدوات، أكثر من 5 لغات برمجة، ساعات من الربط اليدوي
```

<div dir="rtl">

يمنح **steganography-mcp** وكيل الذكاء الاصطناعي الخاص بك 128 أداة عبر 17 فئة من خلال [بروتوكول سياق النموذج](https://modelcontextprotocol.io). يقوم الوكيل بتحليل الصور، فحص JPEG الجنائي، تحليل الصوت، كشف إخفاء النصوص، فحص الملفات الجنائي، تحليل المستندات، تحديد الترميز، تحليل الفيديو والشبكة والأرشيفات، إخفاء GIF وMP3، تحليل الطيف الموزع وBPCS، وإنشاء وتضمين البيانات المخفية &mdash; كل ذلك في محادثة واحدة، ويعمل بالكامل محليًا دون أي اعتماد على خدمات خارجية.

</div>

```
مع steganography-mcp:
  أنت: "حلّل صورة تحدي CTF هذه بحثًا عن بيانات مخفية"

  الوكيل: -> img_detect: Chi-square p=0.0001 (تم كشف إخفاء LSB)،
            تحليل RS يقدّر معدل إخفاء 42%، شذوذ في الإنتروبيا
            في الربع السفلي الأيمن
         -> img_lsb_extract: تم استخراج 847 بايت من بتات RGB الأقل أهمية
         -> crypto_detect: البيانات المستخرجة مرمّزة بـ Base64
         -> crypto_decode: فك الترميز أعطى "FLAG{hidden_in_plain_sight_2024}"
         -> img_known_tools: تطابق توقيع مع OpenStego

         "تحتوي الصورة على إخفاء LSB تم تضمينه باستخدام OpenStego.
          اختبار مربع كاي يؤكد استبدال LSB في جميع قنوات RGB الثلاث
          بمعدل إخفاء 42%. الحمولة المخفية مرمّزة بـ Base64
          وتُفك إلى العلم: FLAG{hidden_in_plain_sight_2024}"
```

## البداية السريعة

### الخيار 1: npx (بدون تثبيت)

</div>

```bash
npx -y steganography-mcp
```

<div dir="rtl">

تعمل جميع الأدوات الـ 128 فورًا. بدون مفاتيح API. بدون إعدادات. يعمل بالكامل دون اتصال.

### الخيار 2: bunx (أسرع)

</div>

```bash
bunx steganography-mcp
```

<div dir="rtl">

### الخيار 3: استنساخ المستودع

</div>

```bash
git clone https://github.com/badchars/steganography-mcp.git
cd steganography-mcp
bun install
```

## ما يستطيع وكيل الذكاء الاصطناعي فعله

### حل تحديات CTF

</div>

```
أنت: "حل تحدي الإخفاء الرقمي في CTF — الصورة في challenge.png"

الوكيل: -> img_detect: تحليل مربع كاي يكشف إخفاء LSB
          (p=0.00012)، تحليل RS يقدّر معدل إخفاء 38%،
          شذوذ في الإنتروبيا عند الكتل (12,4)-(15,8)
       -> img_lsb_extract: تم استخراج 1,247 بايت من بتات RGB الأقل أهمية
       -> crypto_detect: البيانات تطابق نمط Base64 (ثقة: 0.95)
       -> crypto_decode: "Q1RGe3N0M2cwX21hc3Rlcn0=" -> "CTF{st3g0_master}"
       -> img_known_tools: تم كشف توقيع OpenStego في مقاطع PNG

       "تم العثور على العلم: CTF{st3g0_master}
        الصورة تستخدم إخفاء LSB تم تضمينه بواسطة OpenStego.
        الحمولة كانت مرمّزة بـ Base64 في البتات الأقل أهمية
        لقنوات RGB بمعدل إخفاء 38%."
```

<div dir="rtl">

### التحقيق الجنائي الرقمي

</div>

```
أنت: "حلّل suspect.png و suspect.wav بحثًا عن اتصالات سرية"

الوكيل: -> img_detect: لم يتم كشف إخفاء LSB، لكن تم العثور على بيانات ملحقة
          (12,847 بايت بعد علامة PNG IEND)
       -> file_identify: البيانات الملحقة تبدأ بتوقيع ZIP (PK)
       -> img_metadata: مقطع PNG tEXt يحتوي على مفتاح غير عادي "comm"
          مع قيمة Base64
       -> audio_detect: ملف WAV يحتوي على شذوذ مربع كاي في LSB (p=0.003)
       -> audio_silence: تم كشف 3 أقسام صامتة مع بتات LSB نشطة
       -> audio_lsb_extract: تم استخراج 2,048 بايت من بيانات مخفية

       "كلا الملفين يحتويان على بيانات مخفية:
        1. suspect.png يحتوي على أرشيف ZIP ملحق بعد علامة نهاية PNG،
           بالإضافة إلى رسالة مرمّزة بـ Base64 في البيانات الوصفية
        2. suspect.wav يحتوي على إخفاء LSB، تحديدًا في
           الأقسام الصامتة — مؤشر قوي على الإخفاء المتعمد.
        يُوصى باستخراج ZIP وتحليل حمولة WAV."
```

<div dir="rtl">

### كشف النصوص عديمة العرض

</div>

```
أنت: "هذا البريد يبدو عاديًا لكن قد يحتوي على نص مخفي"

الوكيل: -> text_detect: تم كشف 847 حرف عديم العرض بين
          الأحرف المرئية
       -> text_zwc_detect: ZWSP (342)، ZWNJ (505) وُجدت في 127
          موقعًا عبر النص
       -> text_zwc_extract: فك الترميز الثنائي أعطى "EXFIL-ROUTE-7"
       -> text_invisible_scan: 3 أحرف يونيكود غير مرئية إضافية
          (U+2060, U+FEFF) وُجدت في قسم العنوان

       "البريد يحتوي على رسالة مخفية مرمّزة بأحرف عديمة العرض:
        'EXFIL-ROUTE-7'. 847 حرف غير مرئي موزعة عبر النص المرئي
        باستخدام ترميز ZWSP/ZWNJ الثنائي. بالإضافة إلى ذلك، تم
        العثور على 3 أحرف Word Joiner و BOM في العنوان —
        على الأرجح بقايا بيانات وصفية."
```

<div dir="rtl">

### التحليل الجنائي لصور JPEG

</div>

```
أنت: "هل صورة JPEG هذه أصلية أم تم التلاعب بها؟"

الوكيل: -> jpeg_structure: تم تحليل 14 علامة، بنية JFIF طبيعية
       -> jpeg_double_compression: تم كشف ضغط مزدوج —
          درجة تشوه الكتل 0.73، نسبة شذوذ التكميم 0.81
       -> jpeg_quantization: الجودة ~75 (الإضاءة)، لكن الأنماط المتبقية
          تشير إلى أن الأصل كان بجودة ~92
       -> jpeg_exif_deep: EXIF يقول "iPhone 14 Pro" لكن حقل البرنامج
          يقول "Adobe Photoshop 25.0"
       -> jpeg_thumbnail_compare: أبعاد الصورة المصغرة لا تتطابق
          مع الصورة الرئيسية (المصغرة: 4032x3024، الرئيسية: 2048x1536)

       "تم التلاعب بصورة JPEG هذه. الأدلة:
        1. تم كشف ضغط مزدوج — الصورة فُكت وأُعيد ترميزها
           بجودة أقل (الأصلية ~92، الحالية ~75)
        2. EXIF يدّعي iPhone 14 Pro لكن حقل البرنامج يكشف
           تعديل Photoshop
        3. الصورة المصغرة من الالتقاط الأصلي 4032x3024 لكن
           الصورة الرئيسية أُعيد تحجيمها إلى 2048x1536
        جميع النتائج الثلاث تؤكد بشكل مستقل وجود
        تعديل بعد الالتقاط."
```

## استخدام سطر الأوامر

```bash
# عرض المساعدة
npx -y steganography-mcp --help

# سرد جميع الأدوات الـ 128 مع أوصافها
npx -y steganography-mcp --list

# كشف الإخفاء في صورة
npx -y steganography-mcp --tool img_detect '{"file_path":"challenge.png"}'

# استخراج رسالة مخفية من بتات LSB
npx -y steganography-mcp --tool img_lsb_extract '{"file_path":"stego.png"}'

# تحليل مربع كاي
npx -y steganography-mcp --tool img_chi_square '{"file_path":"suspect.png"}'

# تحليل RS (طريقة Fridrich-Goljan-Du)
npx -y steganography-mcp --tool img_rs_analysis '{"file_path":"suspect.png"}'

# كشف الضغط المزدوج لـ JPEG
npx -y steganography-mcp --tool jpeg_double_compression '{"file_path":"photo.jpg"}'

# تحليل EXIF المعمّق
npx -y steganography-mcp --tool jpeg_exif_deep '{"file_path":"photo.jpg"}'

# كشف إخفاء الصوت
npx -y steganography-mcp --tool audio_detect '{"file_path":"message.wav"}'

# كشف ترميز الأحرف عديمة العرض
npx -y steganography-mcp --tool text_zwc_detect '{"text":"suspicious text here"}'

# تضمين رسالة مخفية بأحرف عديمة العرض
npx -y steganography-mcp --tool text_zwc_embed '{"text":"cover text","message":"secret"}'

# تحديد نوع الملف وكشف الملفات المتعددة الصيغ
npx -y steganography-mcp --tool file_polyglot '{"file_path":"suspicious.pdf"}'

# مسح بحثًا عن ملفات مدمجة (بأسلوب binwalk)
npx -y steganography-mcp --tool file_embedded '{"file_path":"mystery.bin"}'

# عرض مرئي للإنتروبيا
npx -y steganography-mcp --tool file_entropy_visual '{"file_path":"data.bin"}'

# كشف تلقائي للترميز
npx -y steganography-mcp --tool crypto_detect '{"input":"aGVsbG8gd29ybGQ="}'

# كسر XOR بالقوة العمياء
npx -y steganography-mcp --tool crypto_xor '{"input":"4f5243484e"}'

# كشف أنماط الشيفرات
npx -y steganography-mcp --tool crypto_patterns '{"input":"Gur dhvpx oebja sbk"}'

# باستخدام Bun (بدء تشغيل أسرع)
bunx steganography-mcp --tool img_detect '{"file_path":"image.png"}'
```

## حالات الاستخدام

### تحديات CTF
حل تحديات الإخفاء الرقمي في مسابقات التقاط العلم. يستطيع وكيل الذكاء الاصطناعي تطبيق جميع تقنيات الكشف بشكل منهجي &mdash; تحليل LSB، فحص البيانات الوصفية، البيانات الملحقة، كشف الترميز، وتحديد الشيفرات &mdash; للعثور على أعلام مخفية في الصور، الملفات الصوتية، المستندات، والنصوص.

### التحقيقات الجنائية الرقمية
كشف قنوات الاتصال السرية في التحقيقات الجنائية. تحليل ملفات المشتبه بهم بحثًا عن بيانات مخفية باستخدام التحليل الإحصائي (مربع كاي، تحليل RS)، فحص البيانات الملحقة بعد علامات EOF، مسح الملفات المدمجة، وتحديد توقيعات أدوات الإخفاء.

### البحث الأمني
تحليل أدوات وتقنيات الإخفاء. مقارنة الصور الأصلية والمعدلة بكسل ببكسل، دراسة توزيعات معاملات DCT في إخفاء JPEG، قياس تغيرات الإنتروبيا من التضمين، والهندسة العكسية لمخططات الترميز.

### التعليم
تعلّم كيف تعمل تقنيات الإخفاء الرقمي. تضمين واستخراج رسائل LSB، ترميز النصوص بأحرف عديمة العرض، عرض مستويات البت وخرائط الإنتروبيا، تحليل بنى الملفات بالتفريغ السداسي العشري، ودراسة أنماط الشيفرات بتحليل التكرار.

### الاستجابة للحوادث
أثناء الاستجابة للحوادث، فحص المستندات والصور بحثًا عن قنوات تسريب مخفية. مسح ملفات PDF بحثًا عن JavaScript مخفي وملفات مدمجة، كشف ترميز الأحرف عديمة العرض في رسائل البريد، تحديد الملفات المتعددة الصيغ، وتحليل الترميزات المشبوهة.

**قرارات التصميم:**

- **4 تبعيات فقط** &mdash; `@modelcontextprotocol/sdk` لبروتوكول MCP، `zod` للتحقق من المدخلات، `pngjs` للوصول إلى بكسلات PNG، `jpeg-js` لفك JPEG. لا شجرة تبعيات منتفخة. لا وحدات أصلية. لا ربط C. لا Python. لا Java.
- **يعمل بالكامل دون اتصال** &mdash; كل أداة تعمل محليًا بالكامل. لا طلبات HTTP. لا استدعاءات API. لا تتبع. لا تبعيات سحابية. ملفاتك لا تغادر جهازك أبدًا.
- **تحليل إحصائي بـ TypeScript خالص** &mdash; اختبار مربع كاي، تحليل RS (Fridrich-Goljan-Du)، تحليل أزواج العينات، إنتروبيا شانون، مؤشر التوافق، وتحليل التكرار — كلها مُنفّذة بـ TypeScript خالص. لا مكتبات رياضيات خارجية.
- **محللات صيغ مخصصة** &mdash; مقاطع PNG، علامات JPEG/EXIF/جداول التكميم، مقاطع RIFF لـ WAV، وترويسات BMP تُحلل بدون أي تبعيات خارجية باستخدام محللات `utils/`. هذا يتيح تحليلًا عميقًا خاصًا بالصيغة لا تستطيع المكتبات العامة تقديمه.
- **17 مزودًا، خادم واحد** &mdash; كل فئة تحليل هي وحدة مستقلة. وكيل الذكاء الاصطناعي يختار الأدوات المناسبة بناءً على سياق التحقيق.
- **نمط ToolDef نظيف** &mdash; كل أداة تتبع نفس النمط `{ name, description, schema, execute }`. إضافة أداة جديدة تتطلب كائنًا واحدًا في الوحدة المناسبة.
- **تحقق Zod في كل حقل** &mdash; كل حقل في المخطط يحتوي على `.describe()` لسياق وكيل الذكاء الاصطناعي. المدخلات غير الصالحة تُلتقط قبل التنفيذ مع رسائل خطأ واضحة.
