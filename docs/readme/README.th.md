## ปัญหา

การซ่อนข้อมูล (Steganography) คือศิลปะของการซ่อนข้อมูลไว้ในที่เปิดเผย &mdash; ภายในรูปภาพ ไฟล์เสียง เอกสาร และแม้แต่ข้อความ Unicode มันถูกใช้ในการแข่งขัน CTF การสืบสวนนิติวิทยาศาสตร์ดิจิทัล ช่องทางการสื่อสารลับ และ payload ของมัลแวร์ การตรวจจับต้องอาศัยการผสมผสานของการวิเคราะห์ทางสถิติ การแยกวิเคราะห์เฉพาะรูปแบบ การวัดเอนโทรปี และความเชี่ยวชาญเฉพาะทาง

```
ขั้นตอนการวิเคราะห์การซ่อนข้อมูลแบบดั้งเดิม:
  ตรวจจับการซ่อนในรูปภาพ       ->  zsteg + stegsolve (2 เครื่องมือ, Ruby + Java)
  วิเคราะห์ไคสแควร์              ->  สคริปต์ Python เฉพาะทาง
  วิเคราะห์ RS                   ->  โค้ด MATLAB/Python เฉพาะทาง
  ตรวจสอบ DCT ของ JPEG           ->  stegdetect (เครื่องมือ C ที่หยุดพัฒนาตั้งแต่ 2004)
  สกัดข้อมูล LSB                 ->  zsteg + steghide + openstego (3 เครื่องมือ)
  ซ่อนข้อมูลในเสียง              ->  Audacity ด้วยมือ + สคริปต์เฉพาะทาง
  ตรวจจับข้อความความกว้างศูนย์    ->  เครื่องมือเว็บ + ตรวจสอบด้วยมือ
  นิติวิทยาศาสตร์ไฟล์ / binwalk  ->  binwalk + foremost + xxd (3 เครื่องมือ)
  ข้อมูล EXIF                    ->  exiftool (ต้องใช้ Perl)
  ตรวจจับการเข้ารหัส             ->  CyberChef เว็บ UI + เดาด้วยมือ
  ─────────────────────────────────
  รวม: มากกว่า 10 เครื่องมือ, มากกว่า 5 ภาษา, หลายชั่วโมงในการเชื่อมโยงผลลัพธ์ด้วยมือ
```

**steganography-mcp** มอบเครื่องมือ 128 รายการใน 17 หมวดหมู่ให้กับ AI agent ของคุณผ่าน [Model Context Protocol](https://modelcontextprotocol.io) เอเจนต์สามารถวิเคราะห์ภาพ ตรวจสอบ JPEG เชิงนิติวิทยาศาสตร์ วิเคราะห์เสียง ตรวจจับการซ่อนข้อมูลในข้อความ วิเคราะห์นิติวิทยาศาสตร์ไฟล์ วิเคราะห์เอกสาร ระบุการเข้ารหัส วิเคราะห์วิดีโอ เครือข่าย และไฟล์บีบอัด ซ่อนข้อมูลใน GIF และ MP3 วิเคราะห์ Spread Spectrum และ BPCS รวมถึงสร้างและฝังข้อมูลที่ซ่อน &mdash; ทั้งหมดในบทสนทนาเดียว ทำงานออฟไลน์ 100% โดยไม่พึ่งพาบริการภายนอก

```
เมื่อใช้ steganography-mcp:
  คุณ: "วิเคราะห์รูปภาพโจทย์ CTF นี้เพื่อหาข้อมูลที่ซ่อนอยู่"

  เอเจนต์: -> img_detect: Chi-square p=0.0001 (ตรวจพบการฝัง LSB),
            วิเคราะห์ RS ประมาณการอัตราการฝัง 42%, ค่าเอนโทรปี
            ผิดปกติที่มุมขวาล่าง
         -> img_lsb_extract: สกัดข้อมูล 847 ไบต์จาก LSB ของ RGB
         -> crypto_detect: ข้อมูลที่สกัดได้เข้ารหัสแบบ Base64
         -> crypto_decode: ถอดรหัสได้ "FLAG{hidden_in_plain_sight_2024}"
         -> img_known_tools: ตรงกับลายเซ็นของ OpenStego

         "รูปภาพมีการซ่อนข้อมูลแบบ LSB ที่ฝังด้วย OpenStego
          การทดสอบไคสแควร์ยืนยันการแทนที่ LSB ในทั้งสามช่อง RGB
          ด้วยอัตราการฝัง 42% ข้อมูลที่ซ่อนอยู่เข้ารหัส Base64
          และถอดรหัสได้เป็นแฟล็ก: FLAG{hidden_in_plain_sight_2024}"
```

## เริ่มต้นอย่างรวดเร็ว

### ตัวเลือกที่ 1: npx (ไม่ต้องติดตั้ง)

```bash
npx -y steganography-mcp
```

เครื่องมือทั้ง 128 รายการใช้งานได้ทันที ไม่ต้อง API key ไม่ต้องตั้งค่า ออฟไลน์ 100%

### ตัวเลือกที่ 2: bunx (เร็วกว่า)

```bash
bunx steganography-mcp
```

### ตัวเลือกที่ 3: โคลน

```bash
git clone https://github.com/badchars/steganography-mcp.git
cd steganography-mcp
bun install
```

### เชื่อมต่อกับ AI agent ของคุณ

## AI ทำอะไรได้บ้าง

### แก้โจทย์ CTF

```
คุณ: "แก้โจทย์การซ่อนข้อมูลใน CTF นี้ — รูปภาพอยู่ที่ challenge.png"

เอเจนต์: -> img_detect: การวิเคราะห์ไคสแควร์ตรวจพบการฝัง LSB
          (p=0.00012), วิเคราะห์ RS ประมาณการอัตราการฝัง 38%,
          เอนโทรปีผิดปกติที่บล็อก (12,4)-(15,8)
       -> img_lsb_extract: สกัดข้อมูล 1,247 ไบต์จาก LSB ของช่อง RGB
       -> crypto_detect: ข้อมูลตรงกับรูปแบบ Base64 (ความเชื่อมั่น: 0.95)
       -> crypto_decode: "Q1RGe3N0M2cwX21hc3Rlcn0=" -> "CTF{st3g0_master}"
       -> img_known_tools: ตรวจพบลายเซ็น OpenStego ใน PNG chunks

       "พบแฟล็ก: CTF{st3g0_master}
        รูปภาพใช้การซ่อน LSB ที่ฝังโดย OpenStego
        ข้อมูลเข้ารหัส Base64 ในบิตที่มีนัยสำคัญน้อยที่สุด
        ของช่อง RGB ด้วยอัตราการฝัง 38%"
```

### การสืบสวนนิติวิทยาศาสตร์ดิจิทัล

```
คุณ: "วิเคราะห์ suspect.png และ suspect.wav เพื่อหาการสื่อสารลับ"

เอเจนต์: -> img_detect: ไม่พบการซ่อน LSB แต่พบข้อมูลต่อท้าย
          (12,847 ไบต์หลังเครื่องหมาย PNG IEND)
       -> file_identify: ข้อมูลต่อท้ายขึ้นต้นด้วยลายเซ็น ZIP (PK)
       -> img_metadata: PNG tEXt chunk มีคีย์ "comm" ที่ผิดปกติ
          พร้อมค่า Base64
       -> audio_detect: ไฟล์ WAV มีค่าไคสแควร์ LSB ผิดปกติ (p=0.003)
       -> audio_silence: ตรวจพบ 3 ส่วนเงียบที่มี LSB ทำงานอยู่
       -> audio_lsb_extract: สกัดข้อมูลที่ซ่อนอยู่ 2,048 ไบต์

       "ทั้งสองไฟล์มีข้อมูลที่ซ่อนอยู่:
        1. suspect.png มีไฟล์ ZIP ต่อท้ายหลังเครื่องหมายสิ้นสุด PNG
           รวมถึงข้อความเข้ารหัส Base64 ในข้อมูลเมตา
        2. suspect.wav มีการซ่อน LSB โดยเฉพาะในส่วนเงียบ
           — ตัวบ่งชี้ที่ชัดเจนของการซ่อนข้อมูลโดยเจตนา
        แนะนำให้สกัดไฟล์ ZIP และวิเคราะห์ข้อมูลใน WAV"
```

### ตรวจจับข้อความความกว้างศูนย์

```
คุณ: "อีเมลนี้ดูปกติ แต่อาจมีข้อความที่ซ่อนอยู่"

เอเจนต์: -> text_detect: ตรวจพบอักขระความกว้างศูนย์ 847 ตัว
```

## การใช้งาน CLI

```bash
# แสดงความช่วยเหลือ
npx -y steganography-mcp --help

# แสดงรายการเครื่องมือทั้ง 128 รายการพร้อมคำอธิบาย
npx -y steganography-mcp --list

# ตรวจจับการซ่อนข้อมูลในภาพ
npx -y steganography-mcp --tool img_detect '{"file_path":"challenge.png"}'

# สกัดข้อความที่ซ่อนอยู่จาก LSB
npx -y steganography-mcp --tool img_lsb_extract '{"file_path":"stego.png"}'

# วิเคราะห์ไคสแควร์
npx -y steganography-mcp --tool img_chi_square '{"file_path":"suspect.png"}'

# วิเคราะห์ RS (วิธี Fridrich-Goljan-Du)
npx -y steganography-mcp --tool img_rs_analysis '{"file_path":"suspect.png"}'

# ตรวจจับการบีบอัดซ้ำ JPEG
npx -y steganography-mcp --tool jpeg_double_compression '{"file_path":"photo.jpg"}'

# วิเคราะห์ EXIF เชิงลึก
npx -y steganography-mcp --tool jpeg_exif_deep '{"file_path":"photo.jpg"}'

# ตรวจจับการซ่อนข้อมูลในเสียง
npx -y steganography-mcp --tool audio_detect '{"file_path":"message.wav"}'

# ตรวจจับการเข้ารหัสอักขระความกว้างศูนย์
npx -y steganography-mcp --tool text_zwc_detect '{"text":"suspicious text here"}'

# ฝังข้อความที่ซ่อนด้วยอักขระความกว้างศูนย์
npx -y steganography-mcp --tool text_zwc_embed '{"text":"cover text","message":"secret"}'

# ระบุประเภทไฟล์และตรวจจับโพลีกลอต
npx -y steganography-mcp --tool file_polyglot '{"file_path":"suspicious.pdf"}'

# สแกนหาไฟล์ที่ฝังอยู่ (แบบ binwalk)
npx -y steganography-mcp --tool file_embedded '{"file_path":"mystery.bin"}'

# แสดงเอนโทรปีแบบภาพ
npx -y steganography-mcp --tool file_entropy_visual '{"file_path":"data.bin"}'

# ตรวจจับการเข้ารหัสอัตโนมัติ
npx -y steganography-mcp --tool crypto_detect '{"input":"aGVsbG8gd29ybGQ="}'

# บังคับ XOR
npx -y steganography-mcp --tool crypto_xor '{"input":"4f5243484e"}'

# ตรวจจับรูปแบบรหัส
npx -y steganography-mcp --tool crypto_patterns '{"input":"Gur dhvpx oebja sbk"}'

# ใช้ Bun (เริ่มต้นเร็วกว่า)
bunx steganography-mcp --tool img_detect '{"file_path":"image.png"}'
```

## กรณีการใช้งาน

### การแข่งขัน CTF
แก้โจทย์การซ่อนข้อมูลในการแข่งขัน capture-the-flag AI agent สามารถใช้เทคนิคตรวจจับทั้งหมดอย่างเป็นระบบ &mdash; วิเคราะห์ LSB, ตรวจสอบข้อมูลเมตา, ข้อมูลต่อท้าย, ตรวจจับการเข้ารหัส, และระบุรหัส &mdash; เพื่อค้นหาแฟล็กที่ซ่อนอยู่ในภาพ ไฟล์เสียง เอกสาร และข้อความ

### นิติวิทยาศาสตร์ดิจิทัล
ตรวจจับช่องทางการสื่อสารลับในการสืบสวนนิติวิทยาศาสตร์ วิเคราะห์ไฟล์ผู้ต้องสงสัยเพื่อหาข้อมูลที่ซ่อนอยู่ด้วยการวิเคราะห์สถิติ (ไคสแควร์, วิเคราะห์ RS), ตรวจสอบข้อมูลต่อท้ายหลังเครื่องหมาย EOF, สแกนหาไฟล์ที่ฝังอยู่, และระบุลายเซ็นเครื่องมือซ่อนข้อมูล

### การวิจัยด้านความปลอดภัย
วิเคราะห์เครื่องมือและเทคนิคการซ่อนข้อมูล เปรียบเทียบภาพต้นฉบับและภาพที่ซ่อนข้อมูลพิกเซลต่อพิกเซล ศึกษาการกระจายค่าสัมประสิทธิ์ DCT ในการซ่อนข้อมูล JPEG วัดการเปลี่ยนแปลงเอนโทรปีจากการฝัง และวิศวกรรมย้อนกลับรูปแบบการเข้ารหัส

### การศึกษา
เรียนรู้วิธีการทำงานของเทคนิคการซ่อนข้อมูล ฝังและสกัดข้อความ LSB, เข้ารหัสข้อความด้วยอักขระความกว้างศูนย์, แสดงระดับบิตและแผนที่เอนโทรปี, วิเคราะห์โครงสร้างไฟล์ด้วย hex dump, และศึกษารูปแบบรหัสด้วยการวิเคราะห์ความถี่

### การตอบสนองต่อเหตุการณ์
ในระหว่างการตอบสนองต่อเหตุการณ์ ตรวจสอบเอกสารและภาพเพื่อหาช่องทางแอบนำข้อมูลออก สแกน PDF เพื่อหา JavaScript ที่ซ่อนอยู่และไฟล์ที่ฝังอยู่, ตรวจจับการเข้ารหัสอักขระความกว้างศูนย์ในอีเมล, ระบุไฟล์โพลีกลอต, และวิเคราะห์การเข้ารหัสที่น่าสงสัย

```
  text/                       # เครื่องมือข้อความ & Unicode (10)
  file/                       # เครื่องมือนิติวิทยาศาสตร์ไฟล์ (10)
  document/                   # เครื่องมือวิเคราะห์เอกสาร (5)
  crypto/                     # เครื่องมือการเข้ารหัส & Crypto (7)
  jpegadv/                    # เครื่องมือวิเคราะห์ JPEG ขั้นสูง (7)
  video/                      # เครื่องมือซ่อนข้อมูลในวิดีโอ (8)
  gif/                        # เครื่องมือซ่อนข้อมูลใน GIF (8)
  network/                    # เครื่องมือซ่อนข้อมูลในเครือข่าย (8)
  mp3/                        # เครื่องมือซ่อนข้อมูลใน MP3 (7)
  spread/                     # เครื่องมือ Spread Spectrum (5)
  bpcs/                       # เครื่องมือวิเคราะห์ BPCS (5)
  archive/                    # เครื่องมือซ่อนข้อมูลในไฟล์บีบอัด (7)
  create/                     # เครื่องมือสร้าง & ฝัง (7)
  qrcode/                     # เครื่องมือซ่อนข้อมูลใน QR Code (6)
  data/
    encoding-patterns.ts      # รูปแบบ regex การเข้ารหัส + ตัวถอดรหัส
    magic-bytes.ts            # ฐานข้อมูลลายเซ็นไฟล์ (มากกว่า 100 รูปแบบ)
    stego-signatures.ts       # ลายเซ็นเครื่องมือซ่อนข้อมูลที่รู้จัก
    unicode-invisible.ts      # ฐานข้อมูลอักขระ Unicode ที่มองไม่เห็น
```

**การตัดสินใจออกแบบ:**

- **4 dependencies เท่านั้น** &mdash; `@modelcontextprotocol/sdk` สำหรับโปรโตคอล MCP, `zod` สำหรับตรวจสอบอินพุต, `pngjs` สำหรับเข้าถึงพิกเซล PNG, `jpeg-js` สำหรับถอดรหัส JPEG ไม่มี dependency tree ที่ใหญ่เกินไป ไม่มี native modules ไม่มี C bindings ไม่มี Python ไม่มี Java
- **ออฟไลน์ 100%** &mdash; ทุกเครื่องมือทำงานบนเครื่องทั้งหมด ไม่มีคำขอ HTTP ไม่มีการเรียก API ไม่มี telemetry ไม่มี cloud dependencies ไฟล์ของคุณไม่เคยออกจากเครื่อง
- **การวิเคราะห์สถิติด้วย TypeScript ล้วน** &mdash; ทดสอบไคสแควร์, วิเคราะห์ RS (Fridrich-Goljan-Du), วิเคราะห์คู่ตัวอย่าง, เอนโทรปีแชนนอน, Index of Coincidence และวิเคราะห์ความถี่ ทั้งหมดเขียนด้วย TypeScript ล้วน ไม่มีไลบรารีคณิตศาสตร์ภายนอก
- **ตัวแยกวิเคราะห์รูปแบบที่สร้างเอง** &mdash; PNG chunks, JPEG markers/EXIF/ตาราง quantization, WAV RIFF chunks และ BMP headers แยกวิเคราะห์โดยไม่มี dependencies ภายนอกโดยใช้ตัวแยกวิเคราะห์ใน `utils/` ทำให้สามารถวิเคราะห์เฉพาะรูปแบบเชิงลึกที่ไลบรารีทั่วไปไม่สามารถทำได้
- **17 providers, 1 เซิร์ฟเวอร์** &mdash; แต่ละหมวดหมู่การวิเคราะห์เป็นโมดูลอิสระ AI agent เลือกเครื่องมือที่จะใช้ตามบริบทของการสืบสวน
- **รูปแบบ ToolDef ที่สะอาด** &mdash; ทุกเครื่องมือใช้รูปแบบเดียวกัน `{ name, description, schema, execute }` การเพิ่มเครื่องมือใหม่คือออบเจ็กต์เดียวในโมดูลที่เหมาะสม
- **ตรวจสอบ Zod ทุกฟิลด์** &mdash; ทุกฟิลด์ใน schema มี `.describe()` สำหรับบริบท AI agent อินพุตที่ไม่ถูกต้องถูกจับก่อนการทำงานพร้อมข้อความแสดงข้อผิดพลาดที่ชัดเจน
